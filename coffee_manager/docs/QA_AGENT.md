# AI Test Agent And QA Dashboard

## מטרה

המשחק כולל בוט בדיקה ויזואלי שמפעיל את המשחק בזמן אמת כדי לזהות בעיות איזון, באגים ותקלות התקדמות.

הכפתורים:

- `הפעל בוט בדיקה (Autopilot)`
- `סיכום משחק והפקת דו"ח`

## איך הבוט אמור לשחק

ב-Stand Mode:

- קונה פולים אם inventory נמוך.
- קונה חלב אם inventory נמוך.
- משרת לקוחות בעזרת פעולה שמדמה Space.
- קונה שדרוגים כשיש כסף.
- לוקח הלוואה רק כשצריך.
- לא מתחיל תחרות עד שכל משימות השלב הושלמו.

בתחרות:

- משרת מהר מאוד כשיש לקוחות בתור.
- משנה מחיר כל כמה שניות.
- מפעיל יכולת שיווק/Ad Pulse.
- לא קונה מלאי.
- לא קונה שדרוגים.

## זרימת שלבים אוטומטית

אם `qa.active === true` והשחקן מנצח תחרות:

1. לשמור כוכבים ושיא.
2. לפתוח שלב הבא.
3. להגדיל `stageIndex`.
4. לקרוא `resetStage(true)`.
5. להחזיר ל-Stand Mode.
6. לדלג על Stage Complete ועל Level Map.

המטרה היא stress-test רציף לכל 5 השלבים.

## דוח QA

הדוח מחולק ל:

- באגים שנמצאו.
- בעיות איזון וקושי.
- הצעות לשיפור.
- דיבאג לשלבים מתקדמים.
- נתוני פעולות הבוט.

צריך לכלול תמיד שורה של התקדמות שלבים:

```text
שלבים שנבדקו בהצלחה: שלב X -> שלב Y
```

## בדיקות חשובות

### Robot Upgrade Verification

אם `Bot Speed` נקנה, זמן השירות חייב לרדת בפועל.

אם `Bot Precision` נקנה, סיכוי בזבוז/שריפה חייב לרדת בפועל.

אם השדרוג רק משנה גרפיקה ולא משנה לוגיקה, הדוח צריך לדווח BUG.

### 20-Second Expiration Audit

כל lot של מלאי צריך להתקלקל אחרי 20 שניות בדיוק.

אם מלאי מתקלקל לפני 19.95 שניות או אחרי 20.12 שניות, הדוח צריך לדווח BUG.

### Bankruptcy & Loan Loop Check

בשלבים מתקדמים הבוט מנסה לקחת כמה הלוואות ולקנות הרבה מלאי כדי לבדוק:

- כסף שלילי.
- חוב שלילי.
- soft-lock.
- חוב שלא ניתן לצאת ממנו.

## תקלות עבר שהיו חשובות

- הבוט לא שירת בתחרות, ולכן הדוכן ננעל.
- הבוט נתקע בשלב 1 ולא עבר אוטומטית.
- כפתור התחרות נשאר נעול בגלל משימות hardcoded נסתרות.
- הדוח צריך לזהות אם שדרוגי רובוט הם visual-only.
